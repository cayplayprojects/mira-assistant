## 📥 Установка и Запуск

Для работы программы потребуется **Python 3.10** или выше.

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/CayPlayProjects/Mira-assistant
cd mira-ai-assistant

```

### Шаг 2: Установка зависимостей

```bash
pip install -r requirements.txt
```

> ⚠️ **Важно для Windows (PyAudio):** Если при установке `pyaudio` возникают ошибки:
> ```bash
> pip install pipwin
> pipwin install pyaudio
> ```

### Шаг 3: Установка Локального AI (Ollama + Qwen 3)

MIRA использует **Ollama** для запуска нейросети.

1.  Скачайте и установите Ollama: [https://ollama.com/download](https://ollama.com/download)
    *   *Windows:* `irm https://ollama.com/install.ps1 | iex` (PowerShell)
    *   *macOS/Linux:* `curl -fsSL https://ollama.com/install.sh | sh`
2.  Запустите терминал и скачайте модель Qwen 3:
    ```bash
    ollama pull qwen3
    ```
    *(Размер ~4-8 ГБ, отличное понимание русского языка)*
3.  Проверьте установку:
    ```bash
    ollama list
    ```
    Должна быть видна модель `qwen3:latest`

### Шаг 4: Запуск МИРА

Убедитесь, что Ollama работает в фоне (иконка 🦙 в трее), и запустите ассистента:

```bash
python mira.py
```

При первом запуске автоматически создастся файл `mira_config.json` с настройками.

### 📋 Управление процессами

В боковом меню есть вкладка **📋 Процессы**, где отображаются все запущенные приложения:
- **Зелёная точка** — приложение активно
- **Кнопка ✕** — закрыть приложение
- **Поле ввода** — быстрое закрытие по названию
- **Кнопка "Обновить"** — обновить список

Также можно закрывать приложения через чат:
- `Закрой блокнот`
- `Выключи Telegram`
- `Останови браузер`

### 📜 Сценарии (Автоматизация)

Вы можете автоматизировать запуск сразу нескольких приложений.

#### Создание сценария:
1. Скажите или напишите: *"Создай сценарий: Стрим"*
2. В открывшемся окне введите название и команды (каждая с новой строки):
   ```text
   obs64.exe
   discord.exe
   steam.exe
   chrome.exe
   ```
3. Нажмите **"Сохранить"**

#### Запуск сценария:
- Скажите: *"Запусти сценарий: Стрим"*
- Или нажмите кнопку ▶ в панели "Сценарии"

#### Удаление сценария:
- Нажмите кнопку ✕ рядом со сценарием

## ⚙️ Настройка (mira_config.json)

Вы можете вручную редактировать файл `mira_config.json`:

### AI Модель
```json
"ai": {
    "base_url": "http://localhost:11434",
    "model": "qwen3",
    "max_tokens": 4096,
    "temperature": 0.4
}
```

### Aliases (Псевдонимы)
Добавьте свои приложения для быстрого запуска:
```json
"aliases": {
    "фотошоп": "photoshop.exe",
    "моя игра": "C:\\Games\\MyGame\\game.exe",
    "иде": "code.exe"
}
```

### Поисковая система
```json
"default_search": "google"
```
Доступны: `yandex`, `google`, `duckduckgo`

### Голос
```json
"voice": {
    "language": "ru-RU",
    "speed": 160
}
```

### Системные команды
```json
"commands": [
    {"trigger": "выключи компьютер", "action": "shutdown /s /t 30", "type": "system"},
    {"trigger": "перезагрузи компьютер", "action": "shutdown /r /t 30", "type": "system"},
    {"trigger": "заблокируй компьютер", "action": "rundll32.exe user32.dll,LockWorkStation", "type": "system"}
]
```

### Контакты
```json
"contacts": {
    "telegram": "@CayPlay78",
    "vk": "https://m.vk.com/cayplay"
}
```

## ❓ Частые вопросы (FAQ)

**Q: МИРА пишет "Qwen 3 не найдена в Ollama".**
**A:** Убедитесь, что вы установили модель:
```bash
ollama pull qwen3
ollama list
```
Если модель установлена, но не определяется — перезапустите Ollama.

**Q: Не работает микрофон.**
**A:** Проверьте:
1. Микрофон подключён и работает в системе
2. Настройки конфиденциальности Windows: *Параметры → Конфиденциальность → Микрофон → Разрешить приложениям доступ*
3. Правильно ли установлен PyAudio:
   ```bash
   pip install pipwin
   pipwin install pyaudio
   ```
4. Если recognizer не ловит голос, попробуйте вызвать `recognizer.adjust_for_ambient_noise(source)` перед записью или уменьшите `energy_threshold`.

**Q: Программа не находит приложение.**
**A:** МИРА теперь показывает подробные сообщения:
- `❌ Не могу найти приложение «Майнкрафт»` — проверьте название
- `💡 Ожидаемый файл: minecraft.exe` — для алиасов
- `💡 Попробуйте запустить через Steam` — для Steam-игр

Решение: откройте `mira_config.json` и добавьте путь в раздел `"aliases"`.

**Q: Как обновить модель ИИ?**
**A:** Выполните в терминале:
```bash
ollama pull qwen3
```

**Q: МИРА использует старую модель.**
**A:** Конфигурация автоматически обновляется. Проверьте `mira_config.json` — в `"model"` должно быть `"qwen3"`.

**Q: Как свернуть МИРУ в трей?**
**A:** Нажмите **✕** в заголовке окна или кнопку **─**. МИРА свернётся в системный трей. Для восстановления дважды кликните по иконке.

**Q: Где хранятся логи?**
**A:** В файле `mira.log` в папке с программой.

## 🐛 Известные проблемы

- **PyAudio** может не устанавливаться на некоторых системах — используйте `pipwin`
- **Steam** должен быть установлен для авто-поиска игр
- Для полного функционала требуются права администратора (закрытие некоторых процессов)
- На Raspberry Pi может потребоваться USB-микрофон и явная указка `device_index` для PyAudio
